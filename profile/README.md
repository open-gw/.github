Welcome to Open Gateway (open-gw)
=================================

**Bridging the gap between enterprise API Gateways and modern infrastructure standards.**

We build high-performance, open-source extensions, plugins, and callouts that modernize how API gateways handle telemetry, security, and traffic governance.

Our Mission
--------------

Enterprise API gateways often suffer from vendor lock-in and proprietary observability patterns. Our goal is to provide **"sweat-free," drop-in extensions** that force proprietary gateways to speak modern, open standards like OpenTelemetry (OTLP)---allowing engineering teams to decouple their API layer from their infrastructure backend.

Supported Ecosystems
-----------------------

We design native, highly optimized artifacts for the world's leading API platforms:

-   **Google Cloud Apigee:** Java Callouts and Shared Flows.

-   **Kong Gateway:** Lua Plugins and Go PDK extensions.

-   **Azure API Management (APIM):** C# Policy Fragments.

Featured Projects
--------------------

-   🔭 **[apigee-otel-logger](https://www.google.com/search?q=https://github.com/open-gw/apigee-otel-logger)** *(In Active Development)*: A high-performance, asynchronous Java Callout to natively export OpenTelemetry (OTLP) logs directly from Apigee Message Processors.
-   🔭 **[GatewayDB-mcp](https://www.google.com/search?q=https://github.com/open-gw/gateway-db-mcp)** *(In Active Development)*: A plugin that securely and efficiently expose relational databases (MySQL, PostgreSQL, SQL Server, etc.) as MCP tools for AI agents
